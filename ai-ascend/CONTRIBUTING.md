# Contributing

## Setup
1. Fork the repository.
2. Create a feature branch from `main`.
3. Run the app locally and verify your changes.

## Pull request
1. Keep PRs focused and small.
2. Describe what changed and why.
3. Mention any DB/schema changes.

## Security
- Do not commit `.env` or any API keys.
- Use `server/.env.example` for sample config only.
